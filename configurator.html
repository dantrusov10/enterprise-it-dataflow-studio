Конфигуратор классов решений

Два режима: для менеджера, который понимает паутину — требования по зонам карты (как на схеме). Для менеджера «с нуля» — один абзац скоупа: система предложит зоны и классы. Два ответа на запрос: классы решений (узлы паутины) и бандл — набор продуктов из каталога kb/merged/vendors-by-class.json (полный мастер-файл: npm run merge-landscape → kb/merged/Enterprise_IT_Landscape_merged.xlsx). Ниже — слой вендоров-эвристика до матрицы.

Режим

Зоны требований (= зоны паутины)

Названия зон можно править. Пустые поля не участвуют в скоринге.

Общий контекст (необязательно)

Добавляется ко всем классам как дополнительный скор.

Скоуп одним текстом

Опишите задачу клиента как получится — система оценит зоны паутины по справочнику зон и подберёт классы.

Быстрые темы

Вставка в активное поле (первая зона или абзац в зависимости от режима).
